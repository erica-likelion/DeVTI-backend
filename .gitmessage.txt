이번 커밋에 대한 구체적인 설명을 작성하세요.
무엇을, 왜, 어떻게 변경했는지를 간단히 정리하면 좋습니다.

# ----------------------------
# Commit Convention Guide
# ----------------------------
# 타입(Type) 종류:
# feat:     새로운 기능 추가
# fix:      버그 수정
# docs:     문서 수정
# refactor: 기능 변경 없는 코드 리팩토링
# test:     테스트 코드
# chore:    빌드/패키지 매니저 등 production code 무관
# comment:  주석 추가 및 변경
# remove:   파일/폴더 삭제
# rename:   파일/폴더명 수정
#
# 제목 규칙:
# - 형식: <type>: <제목>
# - 제목은 명령형으로 간결히 작성
# - ex) feat: 매칭 룸 생성 api 추가
#
# 본문 규칙:
# - 변경 이유, 변경 내용, 영향을 받는 부분 등을 서술
# - 커밋 내용이 너무 간단하거나 긴급할 경우에는 생략 가능
# ----------------------------
